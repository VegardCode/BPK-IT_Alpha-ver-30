# BPK-IT Alpha 3.0

Проект для кейса **«Интеллектуальный отбор данных. Конструктор аналитических выборок для бюджетного планирования»**.

Приложение объединяет выгрузки из РЧБ, Соглашений, Госзаказа и БУАУ, связывает их по бюджетной классификации и дает пользователю интерфейс для построения аналитических выборок без SQL и ручной обработки CSV.

Архив проекта и исходных материалов: [Яндекс.Диск](https://disk.yandex.ru/client/disk/БПК-IT_ALPHA-ver.3.0)

## Возможности

- загрузка и нормализация CSV из 4 источников: РЧБ, Соглашения, ГЗ, БУАУ;
- автоматическое определение дат снимков из имен файлов;
- фильтрация по бюджетной классификации: бюджет, КФСР, КЦСР, КВР, КОСГУ, КВФО;
- выбор показателей и периода анализа;
- два режима расчета: последний снимок и сумма за период;
- сравнение двух периодов с расчетом абсолютного и процентного отклонения;
- таблицы, KPI-карточки, графики и экспорт результатов в Excel;
- авторизация тестовых пользователей и сохранение пользовательских данных.

## Технологии

- **Backend**: Node.js 18+, Express
- **Frontend**: Vue 3 через CDN, TailwindCSS через CDN, Chart.js
- **Парсинг данных**: csv-parse, xlsx, ExcelJS
- **Хранение MVP**: in-memory cache + локальные файлы
- **AI-интеграция**: GigaChat, если настроены переменные окружения

## Локальный запуск

Нужны **Node.js 18+** и **npm**.

1. Откройте папку проекта:

```powershell
cd "C:\Users\Rifresh\Desktop\AmurcodeT\BPK-IT_Alpha ver 30"
```

2. Установите зависимости:

```powershell
npm install
```

3. Запустите сервер:

```powershell
npm start
```

4. Откройте приложение в браузере:

```text
http://localhost:3000
```

При успешном запуске в терминале появится сообщение:

```text
[BOOT] Сервер запущен: http://localhost:3000
```

## Запуск в режиме разработки

```powershell
npm run dev
```

В этом режиме Node.js перезапускает сервер при изменениях файлов.

## Если порт 3000 занят

Ошибка вида `EADDRINUSE: address already in use :::3000` означает, что на порту `3000` уже запущен другой процесс. Чаще всего это уже работающий экземпляр этого же сервера.

Варианты решения:

1. Остановить старый сервер в терминале через `Ctrl+C`.
2. Запустить проект на другом порту:

```powershell
$env:PORT=3001
npm start
```

После этого приложение будет доступно по адресу:

```text
http://localhost:3001
```

## Переменные окружения

Основные переменные:

```env
PORT=3000
DATA_DIR=./Кейс_ Интеллектуальный отбор данных (БФТ, Минфин АО)
```

`PORT` задает порт сервера. `DATA_DIR` позволяет указать другой путь к папке с CSV-выгрузками.

Для AI-функций через GigaChat используется файл `gigachat.env`. Если интеграция не нужна, базовый функционал выборок работает без настройки GigaChat.

## Основные API

- `GET /api/health` — состояние сервера и статистика загруженных данных
- `GET /api/indicators` — список доступных показателей
- `GET /api/snapshots` — список доступных дат снимков
- `GET /api/objects?q=&limit=` — поиск объектов бюджетной классификации
- `GET /api/dictionary/:field` — словарь значений по классификатору
- `POST /api/query` — построение аналитической выборки
- `POST /api/compare` — сравнение периодов
- `POST /api/export/xlsx` — экспорт выборки в Excel
- `POST /api/export/compare-xlsx` — экспорт сравнения в Excel

## Сценарий демонстрации

1. Запустить проект командой `npm start`.
2. Открыть `http://localhost:3000`.
3. Выбрать объект анализа через бюджетную классификацию или поиск.
4. Отметить нужные показатели.
5. Выбрать период и стратегию расчета.
6. Нажать «Получить выборку».
7. Проверить KPI, таблицу и график.
8. При необходимости выгрузить результат в Excel.

## Примечания для разработчика

- Серверная точка входа находится в `server/index.js`.
- Активный frontend берется из `BPK-IT_Alpha-ver-30-main/frontend`.
- Парсеры источников лежат в `server/parsers`.
- Бизнес-логика выборок и сравнения периодов находится в `server/queryEngine.js`.
- При изменении формата CSV сначала проверьте соответствующий парсер, затем логику нормализации в хранилище данных.
